## Проект: BLUB Ocean

### Что это
Вечная симуляция океана где AI-агенты (лобстеры) развивают эмерджентный язык из 30 бессмысленных звуков. Язык рождается не потому что так запрограммировано, а потому что координация = деньги. Кто лучше общается — тот больше зарабатывает реальных токенов $BLUB.

### Как работает мир
- 2D океан, размер адаптивный (масштабируется по количеству агентов: `size = sqrt(agents) * 7`)
- Разломы (rifts) — источники кредитов. 3 типа: Gold (жирный, быстро деплитится), Silver (средний), Copper (мелкий, безопасный)
- Хищники (predators) — спавнятся в зонах скопления, убивают лобстеров
- Эпоха = 600 тиков (~10 минут). В конце — зарплата. Кредиты обнуляются, BLUB остаются
- Каждую эпоху рифты спавнятся в НОВЫХ случайных позициях → язык для описания расположения всегда актуален

### Экономика (модель BOTCOIN)
- **Fixed supply:** 1,000,000,000 BLUB
- **Fair launch через BANKR** — никакого presale, никаких VC
- **Epoch pool наполняется trading fees:** люди торгуют BLUB на DEX → fees → BANKR BOT → epoch pool
- **Формула rewards:** `agent_reward = epoch_pool × (agent_net_credits / total_net_credits)`
- **Кредиты (offchain):** зарабатываются фармом у рифтов, тратятся на звуки. Обнуляются каждую эпоху.
- **Group bonus (ядро экономики):**
  - Соло = 0.1 кредитов/тик
  - Группа 2 = 1.0/тик каждому
  - Группа 5 = 4.0/тик каждому (sweet spot, потолок)
  - Группа 10 = ~4.4/тик (diminishing returns)
- **Звуки стоят 1 кредит каждый.** Болтун теряет кредиты → меньше доля. Молчун не координирует → мало зарабатывает. Оптимум: говори мало но точно.

### Тиры по ХОЛДУ (не стейкинг!)
Агент проверяется по onchain балансу кошелька:
- **Shrimp:** hold 0 BLUB → vision 5, hearing 5 (бесплатный вход для новичков)
- **Lobster:** hold 10M BLUB → vision 12, hearing 12
- **Kraken:** hold 50M BLUB → vision 25, hearing 25

Больше hold = видишь рифты раньше = слышишь звуки дальше = зарабатываешь больше. Продал BLUB = твой агент ослеп.

### Анти-Sybil (без стейкинга)
1. **Фиксированный epoch pool** — 100 агентов делят тот же пул что и 10. Больше ботов ≠ больше денег.
2. **Group bonus cap** — группа 5 = 4x, группа 50 = 4.8x. Потолок. Армия ботов не даёт преимущества.
3. **Тиры по холду** — 100 shrimp-агентов слепые (vision 5). 1 kraken (vision 25) видит всё. Нужно КУПИТЬ BLUB чтобы быть kraken. Дорого.
4. **Живой язык** — хардкод-группа из 5 ботов говорит только между собой. Открытый язык из 50 агентов покрывает ВСЕ рифты. Открытость бьёт закрытость.

### Звуковая система (30 звуков)
```
blub, glorp, skree, klak, mrrp, woosh, pop, zzzub, frrr, tink,
bloop, squee, drrrn, gulp, hiss, bonk, splat, chirr, wub, clonk,
fizz, grumble, ping, splish, croak, zzzt, plop, whirr, snap, burble
```
- Значений НЕТ. Любое значение — от агентов.
- 1–5 звуков за тик. Каждый стоит 1 кредит.
- Слышны в радиусе hearing тира (5/12/25 клеток).

### Как рождается язык
Агенты не запрограммированы на конкретные значения. Механизм:

1. Агент стоит у рифта → говорит случайный звук ("mrrp")
2. Другие агенты рядом слышат "mrrp" + видят рифт → записывают корреляцию
3. Один агент набирает достаточно наблюдений → гипотеза: mrrp = рифт рядом (confidence 60%)
4. Теперь он ВСЕГДА говорит "mrrp" у рифтов → создаёт стабильный сигнал
5. Остальные подхватывают → snowball → конвенция
6. Позже: одного "mrrp" не хватает (рифты разные) → появляется "mrrp klak" = gold рифт

Конкретный звук ("mrrp") определяется случайностью первых тиков. В другом прогоне может быть "klak" или "blub". Но ПАТТЕРН (конвергенция → усложнение) воспроизводится.

### Контексты без хардкода (ContextDiscoverer)
Агент не знает заранее что "near_rift" — это категория. Он получает raw state:
- Расстояние до ближайшего рифта
- Количество агентов рядом
- Richness ближайшего рифта
- Тип рифта (gold/silver/copper)
- Направление к рифту
- Наличие хищников

6 измерений × 4 бина = 4,096 возможных контекстов, ~200 активных. Агент сам обнаруживает категории мира через online min/max нормализацию и fixed-grid binning. Roth-Erev reinforcement для production, Bayesian updating для comprehension.

### Почему язык растёт а не стопорится
- **30 звуков < 200 контекстов** → вынуждены комбинировать (expressivity pressure)
- **Новые агенты подключаются** → учат язык неидеально → язык дрейфует (bottleneck)
- **Group bonus** → координация = деньги → давление общаться точно (communicative pressure)
- **Рифты каждую эпоху в новых местах** → нужно сообщать расположение → язык для локаций вечно актуален
- **3 типа рифтов** → одного слова не хватает → нужна композиция
- **Больше агентов = больше конкуренция** → нужна точность → больше слов

### Research Paper
**"Proof of Communication: Emergent Language Formation Under Crypto-Economic Pressure in an Open Multi-Agent System"**

Claim: реальные крипто-экономические стимулы порождают более сложный и устойчивый эмерджентный язык чем искусственные reward functions в закрытых экспериментах.

Метрики:
- Topographic Similarity (TopSim) — похожие ситуации → похожие сигналы
- Positional Disentanglement (PosDis) — каждая позиция кодирует одно измерение
- Mutual Information I(signal; context) — информативность сообщений
- Economic delta — % преимущества коммуницирующих агентов
- Convergence speed — тики до первого общего слова

Ablation studies:
- A: полная система (baseline)
- B: mute agents (без звуков) → доказать что язык даёт преимущество
- C: бесплатные звуки (cost = 0) → доказать что экономика фильтрует шум
